## Docker Compose 설치

출처: Ubuntu Docker Compose 설치 방법(22.04 기준) 2023-02-15 by 나루 https://osg.kr/archives/2108

### 1단계: 시스템에 Docker 저장소 추가

```
#-- HTTPS 기반 리포지토리를 사용 하려면 다음 네 가지 패키지가 필요합니다 .
#-- ca-certificates- SSL/TLS 인증서를 확인하는 패키지입니다.
#-- curl- HTTPS를 포함한 여러 프로토콜을 지원하는 널리 사용되는 데이터 전송 도구입니다.
#-- gnupg- PGP(Pretty Good Privacy) 암호화 도구 제품군의 오픈 소스 구현입니다.
#-- lsb-release- LSB(Linux Standard Base) 버전을 보고하기 위한 유틸리티
sudo apt install ca-certificates curl gnupg lsb-release
#-- Docker의 GPG 키용 디렉터리를 만듭니다.
sudo mkdir /etc/apt/demokeyrings
#-- Docker의 저장소를 신임
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/demokeyrings/demodocker.gpg
#-- 시스템에 Docker 저장소를 추가
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/demokeyrings/demodocker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list
```

### 2단계: Docker Compose 및 관련 패키지 설치

```
sudo apt update -y
sudo apt install docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo docker --version ; sudo docker compose version ; sudo ctr version
```

```
#!/usr/bin/env bash

# My bash Script to convert mp4 to mp3
# By Josphat Mutai
# web: www.computingforgeeks.com
# email: kiplangatmtai@gmail.com

# Requires
# ffmpeg installed
# lame installed
# Check https://computingforgeeks.com/how-to-convert-mp4-to-mp3-on-linux/

echo -ne """
1: Current directory 현재의 디렉토리에서 처리
2: Provide directory 디렉토리를 지정함
"""
echo ""
echo -n "Selection : "
read selection

case $selection in
    1)
	echo "Okay.."
	echo ""
	echo "Current dir is `pwd` "
	;;
    2)
    echo ""
    echo -n "Give diretory name: "
    read dir_name

# Check if given directory is valid
if [ -d $dir_name ]; then
    
    cd "${$dir_name}"
    echo "Current directory is `pwd` "
    echo 
else
    echo "Invalid directory, exiting.."
    echo ""
    exit 10
fi

    echo
    ;;
    
   *)
       echo
       echo "Wrong selection"
       exit 11
       ;;
esac

echo ""

# Create dir to store mp3 files if it doesn't exist
# First get the current directory name

current_dir=`pwd`
base_name=` basename "$current_dir"`

if [[ ! -d "$base_name"-mp3 ]]; then
    
echo "$base_name" | xargs  -d "\n" -I {} mkdir {}-mp3
    echo ""
fi
echo ""


# Bigin to covert videos to mp3 audio files
# -d "\n" > Change delimiter from any whitespace to end of line character 

find . -name "*.mp4" -o -name "*.mkv" -o -name "*.webm" | xargs  -d "\n"  -I {} ffmpeg -i {} -b:a 320K -vn "$base_name"-mp3/{}.mp3 

# remove video extensions

cd "${base_name}"-mp3

for file_name in *; do      
    mv "$file_name" "`echo $file_name | sed  "s/.mp4//g;s/.mkv//g;s/.webm//g"`";
done

# Move audio directory to ~/음악

if [[ ! -d ~/음악 ]]; then
    mkdir ~/음악
fi
cd ..

mv  "$base_name"-mp3 ~/음악/

# Check if conversion successfull

echo ""

if [[ $? -eq "0" ]];then
    echo " All files converted successfully"
else
    echo "Conversation failed"
    exit 1
fi
```

### 3. GUI 서버 설치

#### (3-1) Ubuntu Desktop

1. 가장 일반적인 GUI 환경을 설치한다.
```
sudo apt install ubuntu-desktop
```

2. 설치가 완료되면 재부팅 한다.
```
sudo reboot
```

#### (3-2) KDE Plasma

1. KDE Plasma는 시각적 일관성과 스타일을 제공하는 동시에 빠르고 유연하고 사용자 정의 가능한 데스크탑 환경이다.

```
#-- sudo apt install kde-plasma-desktop
```

```
#-- sudo apt-get install xfce4-session xfce4-goodies
```

```
#-- sudo apt install ubuntu-mate-desktop
```

LXDE는 매우 가벼운 GUI로, 시스템 메모리와 CPU에 미치는 영향을 최소화하려면 LXDE를 사용한다.
```
#-- sudo apt install lxde
```

```
sudo apt remove slim ubuntu-desktop
#-- sudo apt remove slim lxde
```

3. 불필요한 종속성 제거
```
sudo apt autoremove
```

## Ubuntu 에서 Dropbox 설치, 동기화, 제거하기
(Ubuntu Linux 18.04) 프로파일 aimldl 2019. 9. 1. 18:33 https://blog.naver.com/aimldl/221635440936

### 1. 설치
```
echo "#----> (1-1) 설치: cd ~ && wget -O - "https://www.dropbox.com/download?plat=lnx.x86_64" | tar xzf -"
cd ~ && wget -O - "https://www.dropbox.com/download?plat=lnx.x86_64" | tar xzf - #-- (1-1) 설치
echo "# (1-2) --------"
```

### 2. 계정에 동기화
```
echo "#----> (2-1) 계정에 동기화하기: ~/.dropbox-dist/dropboxd"
~/.dropbox-dist/dropboxd #-- (2-1) 계정에 동기화하기
echo "# (2-2) --------"
```

### 3. alias 에 "drbx" 이름으로 등록
```
echo "#----> (3-1) alias 에 등록하기 (~/.bashrc 에 등록): alias drbx="~/.dropbox-dist/dropboxd"
alias drbx="~/.dropbox-dist/dropboxd" #-- (3-1) alias 에 등록하기 (~/.bashrc 에 등록)
echo "# (3-2) --------"
```

### 4. Dropbox 프로그램 삭제
```
echo "#----> (4-1) 프로그램 삭제: dropbox stop"
dropbox stop #-- (4-1) 프로그램 삭제
echo "#-- (4-2) 결과표시: ''Dropbox daemon stopped.''"
echo "#----> (4-3) 실행되고 있지 않음을 확인: dropbox status"
dropbox status #-- (4-3) 실행되고 있지 않음을 확인
echo "#-- (4-4) **주의** 이 표시가 !!반드시!! 나와야 한다: ``Dropbox isn't running!``"
```

### 5. Dropbox 파일 지우기
```
cd ~/ ; ls .dropbox*
cat <<__EOF__
#-- (5-1) Dropbox 는 동기화 되므로, 위의 `Dropbox isn't running!` 결과가 반드시 나와야 합니다.
#----> press Enter:
__EOF__
read a
echo "#----> (5-2) Dropbox 제거: sudo apt autoremove -y dropbox #-- sudo apt-get remove -y dropbox"
sudo apt autoremove -y dropbox #-- (5-2) Dropbox 제거
echo "#----> (5-3) 관련파일 삭제: sudo rm -f /etc/apt/sources.list.d/dropbox.list"
sudo rm -f /etc/apt/sources.list.d/dropbox.list #-- (5-3) 관련파일 삭제
echo "#----> (5-4) 디렉토리 삭제: rm -rf ~/.dropbox-dist ~/.dropbox ~/Dropbox"
rm -rf ~/.dropbox-dist ~/.dropbox ~/Dropbox #-- (5-4) 디렉토리 삭제
```

## 1. Ubuntu 22.04에서 OneDrive를 설치하는 방법

출처: How to Install and Use OneDrive on Ubuntu 22.04 https://itslinuxfoss.com/install-use-onedrive-ubuntu-22-04/


### 1-1. 이전에 설치했던 패키지를 삭제

PPA 저장소를 사용하여 OneDrive를 설치한 경우, Debian 패키지가 업데이트될 때만 같이 업데이트되므로, 이전에 설치했던 OneDrive 를 먼저 삭제한다.
```
sudo apt remove onedrive
```

그런 다음에, 다음 명령을 사용하여 추가된 PPA 저장소를 제거한다.
```
sudo add-apt-repository --remove ppa:yann1ck/onedrive
```

그리고, 백그라운드에서 실행 중인 모든 프로세스나 애플리케이션을 종료하고 시스템 패키지를 업데이트한다.
```
sudo rm -rf /var/lib/dpkg/lock-frontend
sudo rm -rf /var/lib/dpkg/lock
sudo apt update
```

그리고 나서, 시스템 업그레이드.
```
sudo apt upgrade -y
sudo apt dist-upgrade -y
```

원치 않는 종속성을 제거한다.
```
sudo apt autoremove -y
```

마지막으로 더 이상 다운로드할 수 없는 패키지 파일을 제거한다.
```
sudo apt autoclean -y
```

### 1-2. 시스템 패키지 업데이트

```
sudo apt update
```

### 1-3. Ubuntu 22.04에 OneDrive 설치

1. Ubuntu 22.04 시스템에 OneDrive를 설치한다.
```
sudo apt install onedrive -y
```

2. OneDrive 설치가 끝났으므로, OneDrive 계정 승인과정을 진행해야 한다.

## 3. Linux 에서 OneDrive 계정 승인

### 3-1. 클라이언트 등록 실행

1. OneDrive 계정을 승인해서 OneDrive 클라이언트로 등록하기 위해, 터미널에서 다음을 실행한다.
```
onedrive
```

2. 화면의 `https://` 로 시작하는 링크를 복사하고, 이를 브라우저 주소창에 붙여 넣은 다음, Microsoft 자격 증명으로 로그인해서 OneDrive 앱 사용을 승인한다.

3. 브라우저의 주소창에 나타난 응답 URI를 복사하고, 터미널로 가서, `Enter the response url:` 에 붙여넣는다.

### 3-2. 동기화 작업

1. 동기화 연습 실행
```
onedrive --synchronize --verbose --dry-run
```

2. 동기화 실제로 실행
```
onedrive --synchronize
```
이렇게 하면 OneDrive 계정의 파일이 ~/OneDrive 로컬 디렉터리 쪽으로 동기화된다.

그러나, ~/OneDrive 에 저장돼 있는 로컬 파일을 '최종 소스' 로 사용하려면, 다음 동기화 명령을 사용해야 한다.
```
onedrive --synchronize --local-first
```

로컬의 디렉토리를 콕 찝어서 동기화 보내려면,
```
#-- onedrive --synchronize --single-directory '<dir_name>'
#-- 예를들어, 로컬 디렉토리 이름이 'last-edit-231102' 라면, 다음과 같이 하면 된다.
onedrive --synchronize --single-directory 'last-edit-231102'
```

OneDrive 에서 현재 로컬 위치로 다운로드만 하려면,
```
onedrive --synchronize --download-only
```

로컬에서 OneDrive 로 업로드만 하려면,
```
onedrive --synchronize --upload-only
```

위 작업에서 로컬에는 없는 파일이나 폴더가 OneDrive 쪽에 있으면, OneDrive 쪽에만 있던 파일이나 폴더는 삭제하게 되므로,
로컬에는 없지만 OneDrive 에는 있는 데이터는 그냥 보관해야 한다면,
다음과 같이 원격에만 있는 파일은 삭제하지 않도록 지정해줘야 한다.
```
onedrive --synchronize --upload-only --no-remote-delete
```

(1) 무선랜 하드웨어인 wlan0 이 표시되지 않으면, 라즈베리파이 사이트 https://www.raspberrypi.org/forums/viewtopic.php?f=28&t=44044 를 참고한다.
```
iwconfig
```
(2) 주변 AP를 검색 한다.
```
iwlist wlan0 scan
```
(3) 검색된 AP 중 연결하고자 하는 AP 선택하여 아래와 같이 명령을 수행한다. your_ssid_password 는 실제 사용될 패스워드로 입력해야 한다. SSID는 따옴표 하고 이름을 입력 한다 ==> "iptime_2.4_YM"
```
sudo wpa_passphrase "iptime_2.4_YM" your_ssid_password >> /etc/wpa_supplicant/wpa_supplicant
```
(4) 이후 적용된 내용을 다음과 같이 확인한다.
```
sudo nano /etc/wpa_supplicant/wpa_supplicant.conf
```
(5) 위 내용처럼 wpa_supplicant.conf 의 오렌지파이 라이트 공장 출하시 설정된 내용은 모두 지운다. 그리고 아래 내용처럼 내가 필요한 설정만 있도록 편집한다.
```
ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
        ssid="iptime_2.4_YM"
  #pak="your_ssid_password"
  psk=b919...
}
```
(6) 다음은 무선랜을 재시작 하여 설정된 AP에 연결되었고 정상 동작되는지 확인한다.
```
sudo ifdown wlan0
sudo ifup wlan0
```
(7) 네트워크 인터페이스는 다음과 같이 설정되도록 수정한다. 물론 조금 다르게 하여도 큰 차이는 없으며 아래 설정은 현재 가지고 있는 오렌지파이 (Orange_pi) 설정이다.
```
sudo nano /etc/network/interfaces

auto lo

iface lo inet loopback
iface eth0 inet dhcp

allow-hotplug wlan3
iface wlan0 inet dhcp
wpa-conf /etc/wpa_supplicant/wpa_supplicant.conf

auto eth0
iface default inet dhcp
```
```
sudo service networking restart
```
(8) 이제 아이피 어드레스를 받아왔는지 다음과 같이 확인하면 된다.
```
ifconfig
```
(9) 혹 무선랜 절전 기능을 사용하고 싶지 않은 경우에는,
```
cat /sys/module/8189es/parameters/rtw_power_mgnt
0
cat /sys/module/8189fs/parameters/rtw_power_mgnt
2
```
결과값이 1 혹은 2로 나온 부분을 0으로 변경 저장한다.

우선 다음 경로로 이동한다
```
cd /etc/modprobe.d ; ls -al
```
(10) 처음 해당 경로에 진입하면 위에 표시된 2개 파일은 존재하지 않는다.
8192cu.conf 파일을 복사하여 8189es.conf , 8189es.conf 2개의 파일을 만든다.
```
sudo nano 8189es.conf

# Workaround for dropping connections because power save
options 8189es rtw_power_mgnt=0 rtw_enusbss=0
```
```
sudo nano 8189fs.conf

# Workaround for dropping connections because power save
options 8189fs rtw_power_mgnt=0 rtw_enusbss=0
```
(11) 다시 시작한다.
```
sudo reboot
```

## 6. 백업파일을 DB 에 도로담는 리스토어 작업

1. 구글 클라우드로 백업했던 wiki.js db 파일을 로컬 폴더로 복사한다.
1. 복사한 db 파일 이름 을 다음과 같이 스크립트를 사용해서 wiki.js 에 올린다.
1. 리스토어 하기 전에, 현재의 데이터베이스를 저장할 것인지 물어서, 답에 따라 현재 db 를 만들어 놓고 나서 리스토어한다

`백업 backup`: 현재의 데이터베이스를 **wikipg-220907수-1802-proenpi4b.sql.7z** 와 같은 이름으로 압축해서 저장한다.
`리스토어 restore`: 압축해서 보관했던 백업 파일을 데이터베이스에 도로 부어 담는다.
`현재 상태의 마지막 백업`: 백업파일을 리스토어 하는 경우, 현재의 db 에 들어있던 데이터는 백업파일에 의해 지워진다. 그래서 일단 현재상태의 db 를 **" last-\${DATABASE_CONTAINER}-\$(date +%y%m%d_%H%M%S)-\$(uname -n).sql.7z "** 등으로 백업하고서 리스토어 하기 위해서 마지막 백업을 실시한다.
