이제는 Docker Engine만 설치하면 Docker compose를 플러그인으로 설치하면 된다. 그리고 docker-compose 명령어 대신에 docker compose 명령어를 사용하면 된다.

### 2. 기존의 Old 버전 제거하기

```
sudo apt-get remove docker docker-engine docker.io containerd runc
echo "🎶 (2) ---------"
```

### 3. 리포지터리 설정 방법

1. 필수 패키지 설치
```
sudo apt-get update
sudo apt-get install \
    ca-certificates \
    curl \
    gnupg \
    lsb-release
echo "🎶 (3-1) ---------"
```

2. Docker 공식 GPG Key 추가
```
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "🎶 (3-2) ---------"
```

3. Docker 리포지터리 설정
```
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
echo "🎶 (3-3) ---------"
```

### 4. 도커 엔진 설치

1. apt 패키지 인덱스 업데이트
```
sudo apt-get update
echo "🎶 (4-1) ---------"
```

2. Docker Engine 및 docker-compose-plugin 설치
```
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin
echo "🎶 (4-2) ---------"
```

### 5. Docker Engine Upgrade

1. 배포판 패키지 파일 받으러 가기

현재의 리눅스 배포한 확인:
```
lsb_release -cs
echo "🎶 (5-1) ---------"
```

배포 사이트: https://download.docker.com/linux/ubuntu/dists/
배포 사이트 > `pool` > `stable` > 아키텍처 타입 (arm64) > 아래의 4개 패키지를 다운노드한다.

- `containerd.io_<version>_<arch>.deb`
containerd.io_1.6.24-1_arm64.deb
- `docker-ce_<version>_<arch>.deb`
docker-ce_24.0.6-1~ubuntu.22.04~jammy_arm64.deb
- `docker-ce-cli_<version>_<arch>.deb`
docker-ce-cli_24.0.6-1~ubuntu.22.04~jammy_arm64.deb          - `docker-compose-plugin_<version>_<arch>.deb`
docker-compose-plugin_2.21.0-1~ubuntu.22.04~jammy_arm64.deb

2. 다음과 같이 version 과 arch 를 확인해서,
```
sudo dpkg -i ./containerd.io_<version>_<arch>.deb \
  ./docker-ce_<version>_<arch>.deb \
  ./docker-ce-cli_<version>_<arch>.deb \
  ./docker-compose-plugin_<version>_<arch>.deb
```

실행한다.
```
sudo dpkg -i ./containerd.io_1.6.24-1_arm64.deb ./docker-ce_24.0.6-1~ubuntu.22.04~jammy_arm64.deb ./docker-ce-cli_24.0.6-1~ubuntu.22.04~jammy_arm64.deb ./docker-compose-plugin_2.21.0-1~ubuntu.22.04~jammy_arm64.deb
echo "🎶 (5-2a) ---------"
```

도커 버전 확인. (도커 구동중에도 업데이트 된다)
```
sudo docker --version ; sudo docker compose version
echo "🎶 (5-2b) ---------"
```

### 7. docker-compose.yml 파일

```
version: "3"
services:

  db:
    #-- raspi: image: postgres:11-alpine
    image: postgres:14.1-alpine
    environment:
      POSTGRES_DB: wikidb #-- wiki 데이터베이스 이름
      POSTGRES_USER: imwiki #-- wikijs 사용자 이름
      POSTGRES_PASSWORD: wikiam9ho #-- wikijsrocks 비번
    logging:
      driver: "none"
    restart: unless-stopped
    volumes:
      - /home/docker/pgsql:/var/lib/postgresql/data
    container_name:
      pgsql

  wiki:
    #-- raspi: image: requarks/wiki:2
    image: requarks/wiki:2.5
    depends_on:
      - db
    environment:
      DB_TYPE: postgres
      DB_HOST: db
      DB_PORT: 5432
      DB_NAME: wikidb #-- wiki 데이터베이스 이름
      DB_USER: imwiki #-- wikijs 사용자 이름
      DB_PASS: wikiam9ho #-- wikijsrocks 비번
    restart: unless-stopped
    ports:
      - "9900:3000"
    container_name:
      wikijs
```

## Ubuntu 20.04에 Docker Compose를 설치하는 방법

1. How To Install Docker Compose on Ubuntu 20.04 | Step-by-Step by Mantas Levinas November 15th, 2022 https://www.cherryservers.com/blog/how-to-install-and-use-docker-compose-on-ubuntu-20-04

### 1단계: 시스템에 Docker 저장소 추가

```
#-- HTTPS 기반 리포지토리를 사용 하려면 다음 네 가지 패키지가 필요합니다 .
#-- ca-certificates- SSL/TLS 인증서를 확인하는 패키지입니다.
#-- curl- HTTPS를 포함한 여러 프로토콜을 지원하는 널리 사용되는 데이터 전송 도구입니다.
#-- gnupg- PGP(Pretty Good Privacy) 암호화 도구 제품군의 오픈 소스 구현입니다.
#-- lsb-release- LSB(Linux Standard Base) 버전을 보고하기 위한 유틸리티
sudo apt install ca-certificates curl gnupg lsb-release
#-- Docker의 GPG 키용 디렉터리를 만듭니다.
sudo mkdir /etc/apt/demokeyrings
#-- Docker의 저장소를 신임
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/demokeyrings/demodocker.gpg
#-- 시스템에 Docker 저장소를 추가
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/demokeyrings/demodocker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list
```

### 2단계: Docker Compose 및 관련 패키지 설치

```
sudo apt update -y
sudo apt install docker-ce docker-ce-cli containerd.io docker-compose-plugin
docker --version ; docker compose version ; ctr version
```

### 3단계: YAML 파일 생성

```
mkdir ~/composedemo ; cd ~/composedemo ; vi docker-compose.yml
```
```
# Define services for our configuration. Parameters are similar to those defined by Docker at https://docs.docker.com/samples/wordpress/
services:
  # Create the MariaDB service
  db:
    # Specify the image version to pull from Docker Hub
    image: mariadb:10.6.4-focal
    # Run the command below to configure MariaDB to use the mysql_native_password authentication method
    command: '--default-authentication-plugin=mysql_native_password'
    # Define a storage volume for the database
    volumes:
      - database_data:/var/lib/mysql
    # Always restart the container 
    restart: always
    # Define environment variables for the MariaDB root password, database name, user, and user password
    environment:
      - MYSQL_ROOT_PASSWORD=yourstrongrootpassword
      - MYSQL_DATABASE=wordpress
      - MYSQL_USER=cherry
      - MYSQL_PASSWORD=pepperandegg
    # Define the ports to expose to other containers on the same Docker network
    # These ports are NOT exposed externally to the host
    expose:
      - 3306
      - 33060
  # Create the MariaDB service
  wordpress:
    # Use the latest WordPress image from Docker Hub
    image: wordpress:latest
    # Create a storage volume for the WordPress site
    volumes:
      - wordpress_data:/var/www/html
    # Map port 8081 on the host to port 80 on the container
    ports:
      - 8081:80
    # Always restart the container
    restart: always
    # Use environment variables to allow WordPress to communicate with the database service
    environment:
      - WORDPRESS_DB_HOST=db
      - WORDPRESS_DB_USER=cherry
      - WORDPRESS_DB_PASSWORD=pepperandegg
      - WORDPRESS_DB_NAME=wordpress
# Define the top level volumes
volumes:
    database_data:
    wordpress_data:
```

### 4단계: Docker Compose를 사용하여 앱 실행

```
sudo docker compose up -d
```

### 5단계: 앱 테스트

이제 다음 명령을 사용하여 WordPress 및 MySQL 컨테이너가 실행되고 있는지 확인할 수 있습니다.
```
sudo docker compose ps
```

### 6단계: Docker Compose로 일시 중지 및 중지

일시 중지
```
sudo docker compose pause
```

일시중지 해제
```
sudo docker compose unpause
```

중지
```
sudo docker compose stop
```

컨테이너를 중지하고 연결된 네트워크를 제거하려면 다음 명령을 사용합니다.
```
docker compose down
```

### 7단계: 데모 구성 삭제

데모 앱으로 테스트를 마친 후에는 구성 파일을 삭제할 수 있습니다.
```
rm docker-compose.yml
```

(1) 무선랜 하드웨어인 wlan0 이 표시되지 않으면, 라즈베리파이 사이트 https://www.raspberrypi.org/forums/viewtopic.php?f=28&t=44044 를 참고한다.
```
iwconfig
```
(2) 주변 AP를 검색 한다.
```
iwlist wlan0 scan
```
(3) 검색된 AP 중 연결하고자 하는 AP 선택하여 아래와 같이 명령을 수행한다. your_ssid_password 는 실제 사용될 패스워드로 입력해야 한다. SSID는 따옴표 하고 이름을 입력 한다 ==> "iptime_2.4_YM"
```
sudo wpa_passphrase "iptime_2.4_YM" your_ssid_password >> /etc/wpa_supplicant/wpa_supplicant
```
(4) 이후 적용된 내용을 다음과 같이 확인한다.
```
sudo nano /etc/wpa_supplicant/wpa_supplicant.conf
```
(5) 위 내용처럼 wpa_supplicant.conf 의 오렌지파이 라이트 공장 출하시 설정된 내용은 모두 지운다. 그리고 아래 내용처럼 내가 필요한 설정만 있도록 편집한다.
```
ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1

network={
	ssid="iptime_2.4_YM"
  #pak="your_ssid_password"
  psk=b919...
}
```
(6) 다음은 무선랜을 재시작 하여 설정된 AP에 연결되었고 정상 동작되는지 확인한다.
```
sudo ifdown wlan0
sudo ifup wlan0
```
(7) 네트워크 인터페이스는 다음과 같이 설정되도록 수정한다. 물론 조금 다르게 하여도 큰 차이는 없으며 아래 설정은 현재 가지고 있는 오렌지파이 (Orange_pi) 설정이다.
```
sudo nano /etc/network/interfaces

auto lo

iface lo inet loopback
iface eth0 inet dhcp

allow-hotplug wlan3
iface wlan0 inet dhcp
wpa-conf /etc/wpa_supplicant/wpa_supplicant.conf

auto eth0
iface default inet dhcp
```
```
sudo service networking restart
```
(8) 이제 아이피 어드레스를 받아왔는지 다음과 같이 확인하면 된다.
```
ifconfig
```
(9) 혹 무선랜 절전 기능을 사용하고 싶지 않은 경우에는,
```
cat /sys/module/8189es/parameters/rtw_power_mgnt
0
cat /sys/module/8189fs/parameters/rtw_power_mgnt
2
```
결과값이 1 혹은 2로 나온 부분을 0으로 변경 저장한다.

우선 다음 경로로 이동한다
```
cd /etc/modprobe.d ; ls -al
```
(10) 처음 해당 경로에 진입하면 위에 표시된 2개 파일은 존재하지 않는다.
8192cu.conf 파일을 복사하여 8189es.conf , 8189es.conf 2개의 파일을 만든다.
```
sudo nano 8189es.conf

# Workaround for dropping connections because power save
options 8189es rtw_power_mgnt=0 rtw_enusbss=0
```
```
sudo nano 8189fs.conf

# Workaround for dropping connections because power save
options 8189fs rtw_power_mgnt=0 rtw_enusbss=0
```
(11) 다시 시작한다.
```
sudo reboot
```

## 6. 백업파일을 DB 에 도로담는 리스토어 작업

1. 구글 클라우드로 백업했던 wiki.js db 파일을 로컬 폴더로 복사한다.
1. 복사한 db 파일 이름 을 다음과 같이 스크립트를 사용해서 wiki.js 에 올린다.
1. 리스토어 하기 전에, 현재의 데이터베이스를 저장할 것인지 물어서, 답에 따라 현재 db 를 만들어 놓고 나서 리스토어한다

`백업 backup`: 현재의 데이터베이스를 **wikipg-220907수-1802-proenpi4b.sql.7z** 와 같은 이름으로 압축해서 저장한다.
`리스토어 restore`: 압축해서 보관했던 백업 파일을 데이터베이스에 도로 부어 담는다.
`현재 상태의 마지막 백업`: 백업파일을 리스토어 하는 경우, 현재의 db 에 들어있던 데이터는 백업파일에 의해 지워진다. 그래서 일단 현재상태의 db 를 **" last-\${DATABASE_CONTAINER}-\$(date +%y%m%d_%H%M%S)-\$(uname -n).sql.7z "** 등으로 백업하고서 리스토어 하기 위해서 마지막 백업을 실시한다.
