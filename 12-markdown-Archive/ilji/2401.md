### 1: dash 를 bash 로 바꾸기

ubuntu 에서는 터미널의 셀을 dash 로 쓰고 있으므로 이를 bash 로 바꿔 쓰기로 한다.

```
sudo apt-get install language-pack-ko #-- ubuntu 에서 한글 추가"
sudo timedatectl set-timezone 'Asia/Seoul' ; date #-- ubuntu 에서 시각 지정"
sudo apt install p7zip-full mc net-tools openssh-server git #-- 프로그램 추가
sudo ls -al --color /bin/sh
echo "dash dash/sh boolean false" | sudo debconf-set-selections
DEBIAN_FRONTEND=noninteractive
sudo dpkg-reconfigure dash
sudo ls -al /bin/sh #-- ubuntu 에서 dash 를 bash 로 바꾸기
sudo apt update -y && sudo apt upgrade -y #-- 업데이트와 업그레이드
```

위와 동일한 스크립트를 실행하면서, 진행과정을 설명하는 부분을 추가한 것:

1. How to export/import PuTTY sessions list?
Asked 11 years, 2 months ago Modified 10 months ago Viewed 1.3m times
https://stackoverflow.com/questions/13023920/how-to-export-import-putty-sessions-list

### Export

cmd.exe, requires elevated prompt due to regedit:
Only sessions (produces file putty-sessions.reg on the Desktop):
```
regedit /e "%USERPROFILE%\Desktop\putty-sessions.reg" HKEY_CURRENT_USER\Software\SimonTatham\PuTTY\Sessions
```
All settings except ssh keys (produces file putty.reg on the Desktop):
```
regedit /e "%USERPROFILE%\Desktop\putty.reg" HKEY_CURRENT_USER\Software\SimonTatham
```

### Powershell:
Only sessions (produces file putty-sessions.reg on the Desktop):
```
reg export HKCU\Software\SimonTatham\PuTTY\Sessions ([Environment]::GetFolderPath("Desktop") + "\putty-sessions.reg")
```
All settings except ssh keys (produces file putty.reg on the Desktop):
```
reg export HKCU\Software\SimonTatham ([Environment]::GetFolderPath("Desktop") + "\putty.reg")
```

### Import
Double-click on the \*.reg file and accept the import.

2024년 1월 8일, GitHub는 전 세계 모든 고객도 GitHub/ Microsoft 직원 및 파트너만 사용할 수 있었던 GitHub Certifications (자격 인증) 프로그램을 사용할 수 있다고 발표했습니다. . 1월 8일부터 누구나 [ 등록 사이트 ](https://resources.github.com/learn/certifications/) 에 액세스하여 GitHub Certifications 학습 및 인증 시험 준비를 진행할 수 있다.

[ GitHub Certifications are generally available -The GitHub Blog ](https://github.blog/2024-01-08-github-certifications-are-generally-available/)

이용 가능한 인증 프로그램은 이하의 4종.

### 1. GitHub Foundations Certification

GitHub 플랫폼의 기본 개념과 GitHub 제품을 처음 배우는 사람들을 위한. git의 사용방법으로부터 리포지토리 관리, 커밋, 브랜치, 병합, 프로젝트 관리 등의 GitHub 코어 기능을 한가지 학습할 수 있다.

### 2. GitHub Actions Certification

GitHub Actions를 사용하여 소프트웨어 개발 워크플로를 자동화하는 방법을 배웁니다. CD/ CD 파이프라인 구축 방법, 기업용 Github Actions 관리 방법 등을 배울 수 있다.

### 3. GitHub Advanced Security Certification

개발 수명주기의 모든 단계에서 고급 보안 기능을 사용하여 코드를 보호하는 방법을 배웁니다. GitHub Advanced Security (GHAS) 는 GitHub Enterprise만의 기능으로 기업 개발자가 프라이빗 리포지토리에서의 시크릿 스캔, CodeQL에 의한 코드 스캔, 종속관리 관리 등을 평가 및 구성할 수 있게 되어 있다.

### 4. GitHub Administration Certification

조직의 요구에 부응하여 건전하고 견고하고 안전한 GitHub 환경을 유지하는 데 필요한 모든 것을 익힐 수 있습니다.

GitHub Certifications 시험에 합격하면 자격을 증명하는 Credly 배지와 인증서가 발급됩니다.
또한 [ 각 인증 프로그램의 상세한 학습 가이드 ](https://github.com/LadyKerr/github-certification-guide) 가 리포지토리로 준비되어 있습니다.

## Ubuntu 22.04 사용자 이름 변경하기
aeong-dev 2023. 4. 20. 21:18 https://aeong-dev.tistory.com/8

1. 임시 사용자를 만들고 sudo 권한 부여.
```
TEMP_USER_NAME="tmpusr"
OLD_USER_NAME="oldusr"
NEW_USER_NAME="newuser"
sudo adduser ${TEMP_USER_NAME}
# 새로 만든 임시 사용자에게 sudo 권한 부여
sudo adduser ${TEMP_USER_NAME} sudo
```

2. 지금 로그인 한 터미널 창에서 로그아웃 후 생성한 임시 사용자로 로그인.
진행 중인 process가 있으면 안되는데 ssh 연결이 붙어있다는 것 자체가 process가 있다는 것이기 때문에 꼭 임시 사용자로 로그인하고 진행하기

3. 기존 사용하던 사용자 이름으로 진행 중인 모든 process 죽이기.
```
sudo pkill -u ${OLD_USER_NAME} pid
sudo pkill -9 -u ${OLD_USER_NAME}
```

이거 안해주면 process 있다고 다음 스텝 진행이 안됨 아래 문구처럼 에러 뜸
```
usermod : user {OLD_USER_NAME} is currently used by process 16555
```

4. 사용자 이름 변경 및 변경된 사용자 이름으로 홈디렉토리 변경.
```
sudo usermod -l ${NEW_USER_NAME} ${TEMP_USER_NAME}
sudo usermod -d /home/${NEW_USER_NAME} -m ${NEW_USER_NAME}
``` 

5. group name 변경.
```
sudo groupmod -n ${NEW_USER_NAME} ${OLD_USER_NAME}
```
