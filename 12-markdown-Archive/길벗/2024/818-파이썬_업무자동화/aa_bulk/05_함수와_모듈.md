# 51 함수

```
a값, b값
vvvvvvvv
   V
+--V-----------+
| 함수         |
|   c = a + b  |
+---------V----+
          V
        vvvvv
          c
```

## 511. 함수 정의와 호출

> # 함수의 정의 (= 함수를 만든다) definition
> def 함수명(매개변수):
>      실행할 코드
>      return 반환값
> # 함수 호출 call
> 변수 = 함수명(인자)

```
# 함수 정의
def add(a, b):
    c = a + b
    return c
# 함수 호출
result = add(10, 20)
print(result)
```

## 512. 함수의 번역 순서

```
# 함수 정의
        ++++---<<---------+
        ||||              |
def add(a, b): [2]        |
    c = a + b             |
    return c =>> [3]      |
             //           |
# 함수 호출 //           [1]
  +--------++             |
  | [4]                   |
result = add(10, 20)      |
  |          || //        |
  | [5]      ++++++--->>--+
print(result)
```

## 513. 함수의 주요 용어

```
# 함수 정의
        +--+------------ 매개변수 (함수의 내부로 받아들이는 변수)
        |  |
def add(a, b):
    c = a + b
    return c
           |
           +------------- 반환값 (함수가 내주는 값)
# 함수 호출
             ++--++------ 인자 (함수를 불러낼때 건네주는 깂. 함수는 자신의 매개변수로 받는다)
             ||  ||
result = add(10, 20)
print(result)
```

## 514. 함수를 사용하는 이유

비슷한 문장이 반복되면 코드를 만들기 번거롭고 읽기에도 불편 = 가독성이 떨어진다고 말함.
가독성이 떨어진다 = 오류 발생시 그 위치나 수정부분 찾기 어렵다.
비슷한 문장끼리 묶어서 함수로 만들어야 한다.

# 53 모듈

모듈 = 함수를 모아놓은 것이다

## 531. 모듈 불러오는 방법 3-1: `import`

> import 모듈이름

```
import calendar
```

> 모듈이름.함수이름(인자)

```
import calendar
calendar.prcal(2024)
```

## 532. 모듈 불러오는 방법 3-2: `import-as`

> import 모듈이름 as 별명

```
import calendar as cal #-- as 뒤의 별명은 마음대로 지어도 된다
cal.prcal(2024)
```

## 533. 모듈 불러오는 방법 3-3: `from-import`

> from 모듈이름 import 함수명

```
from calendar import prcal
prcal(2024)
```
이 경우, 저 모듈의 모든 함수를 가져오지 않고, 필요한 함수만 가져오기 때문에 프로그램이 가벼워진다.
모듈 사용시 `모듈이름.함수이름()` 으로 쓰지 않고, `함수이름()` 만 써도 된다.

## 534. 모듈 사용법

> 모듈이름.함수이름(인자)
> 모듈이름.변수
> 모듈이름.클래스이름(인자)
> 모듈이름.클래스이름.메서드이름(인자)

클래스 class = 여러가지 메서드를 한데 묶어놓은 것
메서드 method = 한가지 기능을 구현한것

모듈 module = 그 안에 들어있는 함수, 변수, 클래스, 메서드 등을 쓸 수 있다.
